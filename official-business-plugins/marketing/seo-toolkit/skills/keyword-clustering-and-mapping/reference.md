# Keyword Clustering and Mapping — Reference

## keyword-clustering Package: Output Schema

The `keyword-clustering` package (at `C:\Development\JOHN_OCONNOR\keyword-clustering`, CLI: `keyword-cluster`) produces the following output files in the `--output` directory.

---

### clustered_keywords.csv

Every input keyword with its assigned cluster.

| Column | Type | Description |
|---|---|---|
| `keyword` | string | The input keyword (unchanged) |
| `cluster_id` | integer | Numeric cluster ID (0-based) |
| `cluster_label` | string | Human-readable label inferred from top keywords |
| `volume` | integer | From input CSV |
| `difficulty` | integer | From input CSV |
| `intent` | string | From input CSV |
| `parent_topic` | string | From input CSV |
| `similarity_score` | float | Cosine similarity to cluster centroid (0–1) |

---

### page_map.csv

Maps each cluster to the best-matching existing page (if any).

| Column | Type | Description |
|---|---|---|
| `cluster_id` | integer | Cluster ID |
| `cluster_label` | string | Cluster label |
| `mapped_url` | string | URL of the best-matching existing page. Empty if no match. |
| `mapped_title` | string | Title of the matched page |
| `mapping_confidence` | float | Confidence score 0–1 (0.7+ = strong, 0.4–0.7 = weak, < 0.4 = no match) |
| `cluster_volume` | integer | Total volume across all keywords in cluster |
| `keyword_count` | integer | Number of keywords in cluster |

**Mapping confidence interpretation:**
- **≥ 0.7** — Strong match. The page explicitly targets this cluster. Action: optimise.
- **0.4–0.69** — Weak match. The page incidentally covers the cluster. Action: optimise or create a dedicated page.
- **< 0.4** — No match / GAP. No existing page adequately covers this cluster. Action: create.

---

### gap_report.csv

Clusters with no strong page mapping — content creation opportunities.

| Column | Type | Description |
|---|---|---|
| `cluster_id` | integer | Cluster ID |
| `cluster_label` | string | Cluster label |
| `cluster_volume` | integer | Total volume of cluster keywords |
| `keyword_count` | integer | Keywords in cluster |
| `top_keyword` | string | Highest-volume keyword in cluster |
| `top_keyword_volume` | integer | Volume of top keyword |
| `primary_intent` | string | Dominant intent in cluster |
| `difficulty_avg` | float | Average difficulty across cluster keywords |
| `gap_score` | float | Composite priority score (volume × intent weight × 1/difficulty) |

---

### cannibalization_report.csv

Keywords where multiple existing pages compete, risking ranking dilution.

| Column | Type | Description |
|---|---|---|
| `keyword` | string | The keyword with cannibalisation risk |
| `volume` | integer | Keyword monthly volume |
| `competing_url_1` | string | First competing page URL |
| `competing_url_2` | string | Second competing page URL |
| `url_count` | integer | Total number of pages competing |
| `overlap_score` | float | SERP overlap score (0–1; higher = more cannibalisation risk) |
| `recommended_action` | string | Package suggestion: `consolidate`, `redirect`, `differentiate`, `monitor` |

---

### cluster_summary.csv

One row per cluster with aggregate statistics.

| Column | Type | Description |
|---|---|---|
| `cluster_id` | integer | Cluster ID |
| `cluster_label` | string | Human-readable label |
| `keyword_count` | integer | Keywords in cluster |
| `cluster_volume` | integer | Total monthly volume |
| `difficulty_avg` | float | Average difficulty |
| `primary_intent` | string | Dominant intent |
| `top_keyword_1` | string | Highest-volume keyword |
| `top_keyword_2` | string | Second highest |
| `top_keyword_3` | string | Third highest |
| `mapped_url` | string | Best-mapped page URL (from page_map) |
| `mapping_confidence` | float | Mapping confidence score |

---

### recommendations.md

Free-text markdown file generated by the package summarising:
- Cluster quality metrics (silhouette score, inertia for kmeans)
- Top action items as detected by the package
- Any anomalies (singletons, outlier keywords, oversized clusters)

Always incorporate the content of this file into the Anthril report rather than replacing it.

## Clustering Method Selection Guide

| Method | Best For | Requires Cluster Count? | Speed | Notes |
|---|---|---|---|---|
| `kmeans` | General-purpose; balanced clusters | Yes (N) | Fast | Good starting point. Sensitive to initialisation — run 5 seedings (default). |
| `agglomerative` | Hierarchical exploration; dendrogram analysis | Yes (N) | Medium | Better for understanding sub-clusters. Higher memory use. |
| `hdbscan` | Unknown cluster count; noise-tolerant | No (auto) | Medium | Assigns outlier keywords to noise cluster (-1). Best for messy lists. |
| `graph` | SERP-overlap-based clustering | No (auto) | Slow | Most accurate for commercial intent; requires SerpAPI. Not suitable for purely informational lists. |

## Handoff.json Schema

Written to `${CLAUDE_PLUGIN_DATA}/clusters/<slug>/handoff.json`.

```json
{
  "slug": "string — URL-safe identifier derived from business/campaign name",
  "method": "string — kmeans | agglomerative | hdbscan | graph",
  "cluster_count": "integer — actual clusters produced (may differ from requested N)",
  "csv_paths": {
    "keywords": "absolute path to input keywords CSV",
    "clustered_keywords": "absolute path to clustered_keywords.csv",
    "page_map": "absolute path to page_map.csv",
    "gap_report": "absolute path to gap_report.csv",
    "cannibalization_report": "absolute path to cannibalization_report.csv",
    "cluster_summary": "absolute path to cluster_summary.csv"
  },
  "generated_at": "ISO 8601 datetime string"
}
```

Consumed by: `content-brief-generator`, `internal-linking-planner`, `content-gap-analysis`.

## Content Action Decision Matrix

| Page Map Status | Recommended Action | Priority |
|---|---|---|
| Strong match (≥ 0.7), page performing well | Monitor — no action needed | Low |
| Strong match (≥ 0.7), page underperforming | Optimise on-page: title, headings, content depth | High |
| Weak match (0.4–0.69) | Create a dedicated page or significantly expand existing | Medium |
| No match / GAP | Create new content targeting the cluster | Based on volume |
| Multiple pages matched | Investigate cannibalisation; consolidate or differentiate | High (if confirmed) |
