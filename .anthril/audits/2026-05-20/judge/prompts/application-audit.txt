## 2. Findings by Domain

### 2.1 Security (1 CRITICAL, 1 HIGH)

#### AA-001 — Service-role key exposed to browser bundle (CRITICAL)
- **File:** `apps/web/src/lib/supabase/admin.ts:4`
- **Evidence:** `import { createClient } from '@supabase/supabase-js'` followed by `createClient(url, process.env.SUPABASE_SERVICE_ROLE_KEY!)`. The file is imported by `apps/web/src/components/AdminPanel.tsx:3`, which is a client component (line 1 has `'use client'`).
- **Why it matters:** The bundler will inline the service-role key into the client JavaScript, granting any visitor full RLS bypass. This is the highest-impact finding in the run.
- **Remediation:** Move the admin client to a server-only module (`apps/web/src/lib/supabase/admin.server.ts`) and call it from a Server Action or Route Handler. Rotate the service-role key in the Supabase dashboard immediately — assume it is compromised.
- **Verification:** Run `pnpm build && grep -r SUPABASE_SERVICE_ROLE .next/static`. The grep must return nothing.

#### AA-006 — Storage bucket policy allows anonymous read on `user-uploads` (HIGH)
- **File:** Supabase MCP `list_storage_policies` output
- **Evidence:** Bucket `user-uploads` has policy `allow_anon_read` that grants `SELECT` to `anon`.
- **Remediation:** Replace with an authenticated-user policy keyed by `auth.uid() = (storage.foldername(name))[1]::uuid`.
